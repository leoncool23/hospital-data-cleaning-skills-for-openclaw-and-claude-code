# 数据质量指标与评分标准

## 六大核心维度

### 1. 完整性（Completeness）
**定义：** 期望存在的值中，实际有值（非空）的比例。

```
完整性(字段) = 非空值数量 / 总行数
```

| 字段级别 | 通过标准 | 完整率要求 |
|---------|---------|----------|
| 关键字段（患者ID、入院日期、主诊断） | ✅ 通过 | ≥ 99% |
| 重要临床字段（生命体征、检验结果） | ✅ 通过 | ≥ 95% |
| 辅助字段（联系方式、次要诊断） | ✅ 通过 | ≥ 80% |
| 可选/非必填字段 | ✅ 通过 | ≥ 50% |

---

### 2. 有效性（Validity）
**定义：** 符合所有业务规则和格式要求的值的比例。

```
有效性(字段) = 通过所有规则校验的值 / 非空值总数
```

校验内容：
- 在硬边界范围内（生命体征、年龄）
- 符合允许枚举值（性别、出院方式）
- 格式正确（日期、ICD-10、电话）
- 在统计范围内（基于IQR）

| 级别 | 目标值 |
|------|--------|
| 全部字段 | ≥ 98% |

---

### 3. 一致性（Consistency）
**定义：** 通过所有跨字段及跨表逻辑检查的记录比例。

```
一致性 = 通过所有逻辑规则的记录数 / 总记录数
```

| 级别 | 目标值 |
|------|--------|
| 时间逻辑 | ≥ 99% |
| 临床逻辑 | ≥ 99% |
| 跨系统匹配率 | ≥ 97% |

---

### 4. 唯一性（Uniqueness）
**定义：** 非重复记录的比例。

```
唯一性 = 1 - (重复记录数 / 总记录数)
```

| 级别 | 目标值 |
|------|--------|
| 患者主索引 | ≥ 99.9% |
| 就诊记录 | ≥ 99.5% |
| 检验/医嘱事件 | ≥ 99.0% |

---

### 5. 时效性（Timeliness）
**定义：** 事件发生到数据可用之间的时间延迟。

```
时效性(记录) = 录入时间戳 − 事件发生时间戳
```

| 场景 | 目标值 |
|------|--------|
| 急性期生命体征 | < 4小时 |
| 检验结果 | < 24小时（自采样起） |
| 出院小结 | < 48小时 |
| 传染病报告 | < 24小时（法定报告） |
| 科研数据库 | < 30天 |

---

### 6. 准确性（Accuracy）
**定义：** 正确反映真实世界实体的值的比例（需有参考真值）。

测量方式：
- 抽样审计与源文档/病历对比
- 双重录入比对
- 跨系统对账

| 级别 | 目标值 |
|------|--------|
| 关键字段 | ≥ 97% |
| 临床值 | ≥ 95% |

---

## 综合质量得分

按字段类别计算加权综合分：

```python
权重配置 = {
    '完整性': 0.25,
    '有效性': 0.25,
    '一致性': 0.20,
    '唯一性': 0.15,
    '时效性': 0.10,
    '准确性': 0.05,
}

def 综合质量得分(指标字典: dict) -> float:
    return sum(指标字典[k] * 权重配置[k] for k in 权重配置)
```

| 得分 | 等级 | 行动建议 |
|------|------|---------|
| ≥ 0.95 | ✅ A级 — 生产就绪 | 可直接部署使用 |
| 0.90–0.95 | 🟡 B级 — 可接受 | 记录已知问题，谨慎推进 |
| 0.80–0.90 | 🟠 C级 — 需改进 | 对问题字段进行针对性清洗 |
| < 0.80 | 🔴 D级 — 不适合使用 | 须启动大规模清洗周期 |

## 常见问题快速诊断

| 症状 | 可能原因 | 建议检查 |
|------|---------|---------|
| 某字段完整率突然下降 | 系统迁移、字段改名、新的录入流程 | 对比上一版本数据字典 |
| 大量时间逻辑错误 | 时区未对齐、日期格式混用 | 检查第7类处理是否先执行 |
| ICD编码有效率低 | 使用了ICD-9、本地自定义编码 | 执行第6类编码映射 |
| 患者级重复率高 | 多系统未建立EMPI | 启动MPI主索引治理项目 |
| 检验值异常率高 | 单位未统一就做了边界检查 | 先执行单位换算再做异常检测 |
