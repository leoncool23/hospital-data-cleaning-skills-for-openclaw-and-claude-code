# 🏥 医院数据清洗技能（完整版）

## 总览

本技能提供涵盖15大类别的医院及公共卫生数据清洗完整框架，适用于 Python（pandas/PySpark）、SQL 流水线以及多智能体协同编排。

**执行任何清洗任务前，请完整阅读本 SKILL.md，并按需参阅：**
- `references/cleaning-rules.md` — 业务规则与临床参考范围
- `references/encoding-standards.md` — ICD-10、药品编码、单位换算映射表
- `references/quality-metrics.md` — 完整性、有效性、一致性评分标准
- `scripts/` — 各清洗类别的 Python 可执行模块

---

## 核心原则（不可违背）

1. **绝不在原始数据上直接修改。** 始终在副本上操作，原始数据存档备查。
2. **记录每一处变更。** 日志须包含：操作前后行数、字段名、执行动作、原因。
3. **标记而非静默删除。** 优先新增 `_flag` / `_original` 列，而非直接删除数据。
4. **业务规则须经临床验证。** 所有参考范围与逻辑规则，大规模应用前必须获得临床医生或领域专家确认。
5. **PHI/PII 必须脱敏后方可导出。** 任何数据离开安全环境前均须完成去标识化。
6. **清洗是迭代过程。** 通常需要 3–5 轮才能达到可用质量，每轮结束后与业务方对齐。

## 第 0 阶段 — 数据画像（必须最先执行）

清洗前先生成摸底报告。

```python
# scripts/00_profiling.py
import pandas as pd, numpy as np

def profile(df: pd.DataFrame, label: str = "数据集") -> dict:
    report = {
        "标签":     label,
        "行数":     len(df),
        "列数":     len(df.columns),
        "字段类型": df.dtypes.to_dict(),
        "缺失率":   (df.isnull().mean() * 100).round(2).to_dict(),
        "重复行数": int(df.duplicated().sum()),
        "唯一值数": {c: df[c].nunique() for c in df.columns},
    }
    return report
```

**交付物：** 清洗开始前将画像报告保存至 `output/00_画像报告_<时间戳>.json`。

## 第 1 类 — 缺失值处理

### 1.1 缺失类型判断（至关重要）

| 类型 | 定义 | 医疗场景示例 |
|------|------|------------|
| MCAR（完全随机缺失） | 与任何变量无关的随机缺失 | 仪器随机掉线 |
| MAR（条件随机缺失） | 缺失与已观测变量有关 | 门诊患者不做某项检查 |
| MNAR（非随机缺失） | 缺失与该值本身有关 | 重症患者疼痛评分未填 |

### 1.2 按字段优先级选择策略

| 字段类别 | 缺失比例 | 处理策略 |
|---------|---------|---------|
| 患者ID / 就诊号 | 任意 | 阻断——上报数据所有者 |
| 关键临床字段（生命体征、主诊断） | 任意 | 标记 + 人工审核 |
| 连续型检验值 | < 20% | 分组中位数填充（按科室+诊断分组） |
| 连续型检验值 | 20%–50% | MICE 多重插补 |
| 连续型检验值 | > 50% | 删除该列并记录原因 |
| 分类字段 | < 30% | 众数填充或标记为"未知" |
| 非必填行政字段 | 任意 | 保留空值，添加 `_缺失标记` 指示变量 |

### 1.3 添加缺失指示变量

```python
# MNAR 字段：插补前先添加二值指示变量
df['白细胞_原始缺失'] = df['wbc'].isnull().astype(int)
```

### 1.4 医疗特有规则

> ⚠️ **"未检测 ≠ 缺失"** — 若某检验项目未开具医嘱，其缺失具有临床意义，**不得插补**，应标记为 `未开医嘱`。

## 第 2 类 — 异常值与错误值处理

### 2.1 硬边界规则（临床）

参考范围详见 `references/cleaning-rules.md`。先执行硬边界过滤：

```python
生命体征边界 = {
    "体温_摄氏度":   (34.0, 43.0),
    "心率":          (20,   300),
    "收缩压":        (40,   300),
    "舒张压":        (20,   200),
    "血氧饱和度":    (50,   100),
    "血糖":          (0.5,  55.0),
    "年龄_岁":       (0,    120),
    "体重_千克":     (0.3,  500),
    "身高_厘米":     (30,   250),
}
```

### 2.2 统计检测（IQR 法）

临床数据使用 k=3.0（较通用数据更宽松）：

```python
Q1, Q3 = df[字段].quantile(0.25), df[字段].quantile(0.75)
IQR = Q3 - Q1
异常掩码 = (df[字段] < Q1 - 3.0 * IQR) | (df[字段] > Q3 + 3.0 * IQR)
```

### 2.3 异常值子类分类（不直接删除，先分类标记）

| 标记值 | 含义 |
|-------|------|
| `硬边界` | 超出生理可能范围——疑似录入错误 |
| `统计异常` | 统计学极端——临床上可能真实存在 |
| `单位错误` | 疑似单位混用（如磅录成公斤） |
| `小数点漂移` | 值×10 或÷10 后落入正常范围 |

### 2.4 单位统一（异常检测前必须先统一单位）

完整换算表见 `references/encoding-standards.md`。

## 第 3 类 — 小样本抑制（公卫重点）

所有汇总公共卫生输出均需执行此步骤。

- 默认阈值：**n < 5** → 抑制（替换为 `<5`）
- 二次抑制：若抑制一个单元格后，通过差值可推算出另一个，则同步抑制互补单元格
- 替代方案：地理聚合（区县→地市）、时间聚合（周→月）、差分隐私加噪

> 此类别在**报告生成时**执行，而非清洗阶段。在清洗数据集中用 `_已抑制 = True` 标记小样本行。

### 4.1 三层去重（由粗到细）

**A层 — 完全重复行**（系统同步产生的冗余）

**B层 — 患者级重复**（同一患者多次建档）

# 使用 recordlinkage 库按姓名+出生日期+性别做概率匹配

## 第 6 类 — 编码与术语标准化

### 6.1 诊断编码（ICD-10）

1. 格式校验：`^[A-Z]\d{2}(\.\d{1,4})?$`
2. 对照官方 ICD-10-CM 标准码表核实（存储于 `references/encoding-standards.md`）
3. 历史数据 ICD-9 → ICD-10 转换
4. 原始编码保存至 `ICD编码_原始` 列

### 6.2 手术操作编码（ICD-9-CM-3 / ICD-10-PCS）

同上校验逻辑。自由文本手术名称标记为待人工编码。

### 6.3 药品标准化

优先级顺序：
1. 院内药品编码 → ATC 编码
2. 商品名 → 通用名（INN）
3. 剂量单位统一 → mg（详见换算表）
4. 标记同类不同名药品的多重用药冲突

### 6.4 核心标准参考

| 领域 | 标准 |
|------|------|
| 诊断 | ICD-10-CM（门诊）/ ICD-10（住院） |
| 手术操作 | ICD-10-PCS / SNOMED-CT |
| 检验项目 | LOINC |
| 药品 | ATC / WHO INN |
| 微生物 | SNOMED-CT |
| 术语集 | UMLS 元叙词表 |

## 第 7 类 — 时间数据处理

### 7.1 所有时间戳统一为 ISO 8601 格式

```python
from dateutil import parser as du_parser

def 安全解析日期(s, 格式='%Y-%m-%d'):
    try:
        return du_parser.parse(str(s)).strftime(格式)
    except Exception:
        return None  # 标记，不猜测
```

### 7.2 时区处理

- 内部统一存储为 UTC
- 原始时区保存至 `_原始时区` 列
- 仅在展示时转换为本地时区

### 7.3 需要计算的衍生时间变量

| 变量 | 计算公式 | 使用场景 |
|------|---------|---------|
| `住院天数` | 出院日期 − 入院日期 | 医疗效率 |
| `确诊延迟天数` | 诊断日期 − 症状发生日期 | 流行病学/临床 |
| `门到针时间_分钟` | 治疗开始 − 到院时间 | 卒中/心梗质控 |
| `随访天数` | 末次就诊 − 出院日期 | 再入院风险 |
| `流行病学周` | 症状发生日期的 ISO 周 | 疫情监测 |
| `报告延迟天数` | 报告日期 − 发病日期 | 公共卫生 |

## 第 8 类 — 隐私脱敏与数据安全

### 8.1 直接标识符（须删除或假名化）

姓名、身份证号、电话、地址、邮箱、病历号、IP地址、设备标识符、完整出生日期（与罕见病+小区域组合时）。

### 8.2 准标识符（需做 k-匿名性检查）

年龄 + 邮政编码/区县 + 罕见诊断 存在重识别风险。处理方式：
- 年龄 → 年龄组（<1、1–4、5–14、15–44、45–64、65+）
- 地理 → 聚合至区/市级
- 日期 → 随机偏移（患者内一致）或截断至月/年

### 8.3 技术方案

| 技术 | 适用场景 |
|------|---------|
| 假名化 | 研究数据链接（主密钥保存于加密存储） |
| k-匿名（k≥5） | 数据集对外发布 |
| 差分隐私 | 汇总统计发布 |
| 数据遮蔽 | 开发/测试环境 |

### 8.4 ⚠️ 日志中严禁出现 PHI

清洗日志、错误报告、审计追踪中不得包含患者姓名、ID 或联系方式，仅使用假名化主键。

## 第 9 类 — 数据结构清洗

### 9.1 字段类型强制规范

```python
类型规范 = {
    "患者ID":    "str",
    "年龄_岁":   "float",
    "入院日期":  "datetime64[ns]",
    "性别":      "category",
    "ICD10编码": "str",
}
for 字段, 类型 in 类型规范.items():
    if 字段 in df.columns:
        df[字段] = df[字段].astype(类型)
```

### 9.2 宽表 ↔ 长表转换

大多数HIS导出为宽表（每行一患者，每列一次就诊）。时间序列分析需转为长表（每行一次观测）：

```python
df_长表 = pd.melt(df_宽表,
    id_vars=['患者ID', '入院日期'],
    value_vars=['白细胞_第1天', '白细胞_第3天', '白细胞_第7天'],
    var_name='测量时间点',
    value_name='白细胞值'
)
```

### 9.3 关系规范化

至少满足第一范式（1NF：原子值、无重复组）。任何有意为之的反规范化须在数据字典中说明。

## 第 10 类 — 文本与 NLP 清洗

适用于自由文本临床记录、主诉、诊断描述等字段。

### 10.1 基础文本规范化

```python
import re
def 规范化文本(text: str) -> str:
    text = str(text).strip()
    text = re.sub(r'\s+', ' ', text)                      # 合并空白
    text = re.sub(r'[^\w\s\u4e00-\u9fff]', '', text)      # 保留汉字+字母数字
    return text.lower()
```

### 10.2 医学缩写扩展

在 `references/encoding-standards.md` 维护缩写词典，例如：
- HTN → 高血压
- DM → 糖尿病
- CAD → 冠心病
- SOB → 呼吸困难

### 10.3 命名实体识别（NER）结构化抽取

使用医疗 NER 模型（spaCy + 自定义临床实体，或 BioBERT / RoBERTa-wwm-ext）抽取：
- 诊断名称及日期
- 药品名称 + 剂量
- 症状描述 + 发生时间
- 手术/操作名称

## 第 11 类 — 数据来源核对

### 11.1 跨系统对账

```

1. 随机抽取 50–100 条记录
2. 逐条追溯至源系统或纸质病历
3. 计算各字段准确率

## 第 14 类 — 数据集成

### 14.1 多系统合并检查清单

```
□ 确认患者ID / EMPI 在所有系统中为同一主键
□ 关联前统一日期格式
□ 检查系统间时区偏差
□ 明确处理一对多关联（如一次住院对应多条检验结果）
□ 每次关联前后审计行数变化
□ 标记仅存在于某一系统的记录
```

### 14.2 冲突解决优先级

当同一字段在多个系统中取值不同时：

1. 以数据字典中指定的权威源系统为准（按字段定义）
2. 时间戳较新的值优先（两系统同等权威时）
3. 上述规则均无法解决 → 标记为待人工审核

├── 04_质量报告_<时间戳>.md            # 人类可读汇总报告
└── 05_数据字典_<时间戳>.xlsx
```

### 15.2 需要报告的质量指标

| 指标 | 计算公式 | 目标值 |
|------|---------|--------|
| 完整性 | 非空值 / 总行数 | 关键字段 > 95% |
| 有效性 | 通过规则校验的值 / 总行数 | > 98% |
| 一致性 | 通过逻辑检查的记录 / 总行数 | > 99% |
| 唯一性 | 1 - 重复率 | > 99.9% |
| 时效性 | 从事件发生到数据录入的延迟 | 急性期 < 48小时 |

### 15.3 版本控制

- 清洗数据集打语义化标签：`v1.0.0_已清洗_<日期>`
- 清洗脚本纳入 Git 管理，提交信息格式：`clean: <类别> - <描述>`
- 每轮清洗后更新数据字典

### 15.4 双人复核制度

公共卫生项目：影响 > 1% 记录的清洗决策须由第二名分析师签字确认。

## Agent 执行工作流

执行完整清洗流水线时，遵循以下顺序：

```
第0阶段  → 原始数据画像
第1阶段  → 结构修复（第9类）— 先处理类型、格式
第2阶段  → 去重（第4类）— 建立唯一键
第3阶段  → 时间清洗（第7类）— 修复日期，派生时间变量
第4阶段  → 缺失值处理（第1类）
第5阶段  → 异常值检测（第2类）
第6阶段  → 逻辑一致性校验（第5类）
第7阶段  → 编码标准化（第6类）
第8阶段  → 文本清洗（第10类）— 如存在自由文本字段
第9阶段  → 脱敏处理（第8类）— 任何数据导出前必须执行
第10阶段 → 来源核对（第11类）— 抽样审计
第11阶段 → 衍生变量构建（第12类）
第12阶段 → 数据集成（第14类）— 多源数据时执行
第13阶段 → 小样本抑制（第3类）— 报告生成阶段执行
第14阶段 → 标准化/缩放（第13类）— 仅在模型训练前执行
第15阶段 → 生成质控报告（第15类）
```

> 在 OpenClaw 多智能体环境中，每个阶段可分配给专职子智能体。计算量最大的第8阶段（去重）和第10阶段（文本NLP）可使用 `sessions_spawn` 并行执行。

## OpenClaw 多智能体配置

将本技能放入 `~/.openclaw/skills/hospital-data-cleaning-zh/` 供所有 Agent 使用，或放入 `<workspace>/skills/hospital-data-cleaning-zh/` 供特定 Agent 使用。

大数据集推荐 Agent 角色分工：

| Agent ID | 职责 | 负责阶段 |
|---------|------|---------|
| `编排者` | 协调流水线，生成最终质控报告 | 第0、15阶段 |
| `结构清洗者` | 类型规范、去重、时间处理 | 第1、2、3阶段 |
| `临床清洗者` | 异常值、逻辑校验、编码标准化 | 第4、5、6阶段 |
| `文本处理者` | 文本清洗、NER、缩写扩展 | 第8阶段 |
| `隐私保护者` | 脱敏、k-匿名性处理 | 第9阶段 |
| `分析师` | 衍生变量构建、数据标准化 | 第11、14阶段 |

`openclaw.json` 子智能体配置示例：

      "skills": ["hospital-data-cleaning-zh"],
